### 3-1. 회원가입 & 결과 페이지

#### 필수 요구사항

- [ ] 결과 페이지에서 입력값을 `table`로 표시

#### 동작 테스트

- [ ] 정상 제출 시 `result.html`의 표에 각 값이 정확히 표시되는지 확인
- [x] 결과 페이지를 스크롤해도 상단 헤더가 고정되는지 확인

### 3-2. 여행지 소개 페이지

#### 필수 요구사항

- [ ] 전체 구조를 `header`, `nav`, `main`, `section`, `article`, `aside`, `footer` 시맨틱 요소로 구성
- [ ] 내비게이션을 `position: fixed`로 고정
- [ ] 내비게이션 링크의 `href="#id"`를 각 `section`의 `id`와 연결

#### 세부 요구사항

- [ ] 명소 갤러리를 `figure` + `figcaption` 3개 이상으로 구성

### 3-3. 반응형 상품 카드 갤러리

#### 필수 요구사항

- [ ] `:root`에 5개 이상의 색상 변수를 정의하고 전 구간에서 `var()` 사용
- [ ] 6개 이상의 상품 카드를 `display: grid` + `repeat(3, 1fr)`로 3열 배치
- [ ] 상단바와 카드 내부 정렬에 Flexbox의 `space-between`, `margin-left: auto` 사용

#### 세부 요구사항

- [ ] `@media`로 900px 이하에서 2열로 변경
- [ ] `@media`로 600px 이하에서 1열로 변경하고 메뉴 숨김
- [ ] 카드 hover 시 `transform: translateY()`와 강화된 그림자 적용
- [ ] hover 효과에 `transition`을 적용하여 부드럽게 전환
- [ ] 히어로 제목·부제에 `@keyframes`와 `delay`를 적용한 순차 등장 애니메이션
- [ ] 다크모드 버튼으로 테마와 버튼 표시(🌙/☀️) 전환

#### 기본 동작 테스트

- [ ] 화면 폭 1200px → 800px → 500px에서 카드가 3열 → 2열 → 1열로 변경되는지 확인
- [ ] 500px에서 상단 메뉴가 숨겨지는지 확인
- [ ] 🌙 다크 버튼 클릭 시 배경·카드·글자색과 버튼이 한 번에 전환되는지 확인
- [ ] 카드 hover 시 카드가 위로 떠오르고 그림자가 진해지는지 확인

#### 검증·결과 테스트

- [ ] 새로고침 시 히어로 제목과 부제가 아래에서 순차적으로 등장하는지 확인
- [ ] 상단바에서 로고는 왼쪽, 메뉴·버튼은 오른쪽으로 정렬되는지 확인
- [ ] OS의 ‘동작 줄이기’를 켜고 새로고침하면 애니메이션이 비활성화되는지 확인

### 3-4. 인터랙티브 할 일 관리 앱

#### 필수 요구사항

- [ ] 추가 버튼 또는 Enter 키로 할 일을 추가하고 빈 값은 방지
- [ ] 체크박스로 완료 상태를 토글하고 완료 항목에 취소선 표시
- [ ] `✕` 버튼으로 할 일 삭제

#### 세부 요구사항

- [ ] 전체 / 진행 중 / 완료 필터 구현
- [ ] 하단에 전체 개수와 완료 개수 요약 표시
- [ ] 부모 `ul`에 리스너를 한 번만 등록하는 이벤트 위임으로 동적 항목 처리

#### 추가 요구사항

- [ ] `storage.js`(데이터) / `app.js`(화면)로 모듈을 분리하고 `type="module"` 사용
- [ ] `localStorage`로 저장·복원하고 오늘의 한마디를 `fetch`로 로드하며 실패 시 기본 문구 표시

#### 기본 테스트

- [ ] Live Server 또는 `python -m http.server`로 실행하여 ES 모듈 오류가 없는지 확인

- [ ] 체크박스 클릭 시 취소선이 표시되고 하단 완료 개수가 1 증가하는지 확인
