## 3. 본인이 직접 정리한 결과물 평가

### 3-1. 회원가입 & 결과 페이지

#### 필수 요구사항

- [x] 이름·아이디·비밀번호·이메일·약관 동의에 `required` 적용
- [x] 아이디에 `pattern`을 적용하여 영문과 숫자 4~12자만 허용
- [x] 비밀번호에 `minlength="8"` 적용
- [x] 전화번호에 `010-0000-0000` 형식의 `pattern` 적용

#### 세부 요구사항

- [x] 성별은 `radio`로 구성하여 하나만 선택
- [x] 관심 분야는 `checkbox`로 구성하여 복수 선택
- [x] 지역은 `select`로 구성
- [x] 상단 헤더를 `position: fixed`와 `z-index`로 고정
- [x] 제출 시 `result.html`로 이동
- [x] 결과 페이지에서 입력값을 `table`로 표시
- [x] 관심 분야 복수 값을 결과에 모두 표시

#### 동작 테스트

- [x] 빈 필수값 제출 시 브라우저 경고와 제출 차단 확인
- [x] 아이디 2자는 패턴 위반, 4자 이상은 통과하는지 확인
- [x] 비밀번호 3자 입력 시 8자 이상 안내 확인
- [x] 관심 분야 “프론트엔드”와 “디자인”이 결과에 함께 표시되는지 확인
- [x] 정상 제출 시 `result.html`의 표에 각 값이 정확히 표시되는지 확인
- [x] 결과 페이지를 스크롤해도 상단 헤더가 고정되는지 확인

#### 자기평가

- 잘된 점:
  1. 입력값 검증을 JavaScript에만 의존하지 않고 `required`, `pattern`, `minlength`, `type`, `autocomplete` 등 HTML이 제공하는 속성을 활용하여 웹 표준에 맞는 회원가입 폼을 구현한 점이 잘되었다고 생각합니다.
  2. `form`, `label`, `input`, `select`, `table` 등의 태그를 목적에 맞게 사용하면서 폼의 제출 과정과 각 요소의 역할을 이해하고, 화면 모양뿐 아니라 문서의 의미가 드러나도록 구조를 작성한 점이 잘되었다고 생각합니다.
  3. 사용자의 입장에서 입력 과정에서 불편할 수 있는 부분을 고민하고, 필수 입력 표시, 입력 조건 안내, 형식 예시, 자동완성, 버튼 hover와 모바일 반응형 디자인을 추가하여 사용 편의성을 높인 점이 잘되었다고 생각합니다.
  4. 개인정보가 URL과 방문 기록에 노출될 가능성을 줄이기 위해 `sessionStorage`로 데이터 전달 방식을 변경하고, 비밀번호 원문을 저장하지 않도록 처리하면서 데이터 전달 과정의 보안 문제를 고민하고 개선한 점이 잘되었다고 생각합니다.
  5. 성별, 관심 분야, 지역의 표시 문구를 상수로 분리하여 반복되는 문자열을 줄이고 수정하기 쉽게 구성한 점이 잘되었다고 생각합니다.
- 어려웠던 점과 해결 과정:
  1. 처음에는 `form`의 `action`으로 결과 페이지에 이동하면서 입력값을 URL 쿼리스트링으로 전달했는데, 이름·이메일·비밀번호와 같은 개인정보가 주소창과 방문 기록에 그대로 노출되는 문제가 있었습니다. 단순 실습이더라도 개인정보 노출 가능성을 줄이고 싶었으며, 제출 이벤트에서 입력값을 `sessionStorage`에 저장한 뒤 결과 페이지에서 꺼내 표시하는 방식으로 변경하는 과정이 어려웠습니다.
  2. `sessionStorage`도 JavaScript로 접근할 수 있어 비밀번호 원문을 저장하는 것은 적절하지 않다고 판단하였습니다. 비밀번호 대신 입력 여부만 `hasPassword`로 저장하고, 결과 화면에는 사용자가 입력한 실제 비밀번호 길이와 관계없이 고정된 별표 8개를 표시하도록 구현하였습니다. 데이터 전달 기능뿐 아니라 어떤 정보를 저장하고 화면에 노출할지 결정하는 부분이 어려웠지만, 민감정보를 다룰 때 고려해야 할 점을 경험할 수 있었습니다.
- 개선할 점:
  1. 기능 구현에 집중해 전체적인 디자인이 단순합니다. 색상과 여백, 입력 요소의 상태별 디자인을 보완하고 싶습니다.
  2. 성별, 관심 분야, 지역의 선택 항목이 HTML에 반복되어 있어 수정할 때 각각 변경해야 합니다. 선택 항목을 JavaScript에서 생성하여 한 곳에서 관리하고 싶습니다.
  3. 현재는 약관 내용을 확인하지 않고 동의할 수 있습니다. 약관의 마지막까지 이동해야 동의할 수 있도록 하고, 약관 확인 과정의 편의성을 높이기 위해 `맨 아래로 이동` 버튼도 제공하고 싶습니다.
  4. 현재는 비밀번호를 한 번만 입력하므로 오타가 있어도 바로 확인하기 어렵습니다. 비밀번호 확인 입력란을 추가하고 두 값의 일치 여부를 검증할 수 있도록 개선하고 싶습니다.
- 실습 이후 추가한 과제/기능:
  1. 사용자 편의성을 고려해 MDN 문서를 참고하여 회원가입 입력 필드에 `autocomplete` 속성을 적용
  2. 일관된 크기 계산과 기본 여백 제거를 위해 전역 CSS 초기 스타일을 적용
  3. 사용자가 필수 입력 항목을 직관적으로 구분할 수 있도록 라벨에 빨간 별표와 필수 입력 안내 문구를 적용
  4. 운영체제별 글꼴 차이를 줄이고 일관된 한글 UI를 제공하기 위해 Google Fonts의 Noto Sans KR을 적용
  5. 개인정보가 URL 쿼리스트링에 노출되는 것을 방지하기 위해 `sessionStorage`로 입력값을 전달하고, 민감정보인 비밀번호는 원문 대신 입력 여부만 저장하여 결과 화면에 고정된 별표(`********`)로 표시하도록 개선
  6. 저장된 회원가입 정보 없이 결과 페이지에 직접 접근하면 안내 메시지를 표시하고 회원가입 페이지로 이동하도록 예외 처리
  7. 사용자가 입력 조건과 형식을 미리 확인할 수 있도록 각 입력란에 안내 문구와 형식 예시를 `placeholder`로 제공

### 3-2. 여행지 소개 페이지

- [x] `video`에 `controls`, `poster`, `source`를 작성

#### 필수 요구사항

- [x] 전체 구조를 `header`, `nav`, `main`, `section`, `article`, `aside`, `footer` 시맨틱 요소로 구성
- [x] 내비게이션을 `position: fixed`로 고정
- [x] 내비게이션 링크의 `href="#id"`를 각 `section`의 `id`와 연결

#### 세부 요구사항

- [x] 명소 갤러리를 `figure` + `figcaption` 3개 이상으로 구성
- [x] 먹거리 섹션을 본문 `article`과 여행 팁 `aside`의 2단으로 배치

#### 추가 요구사항

- [x] `video`에 `controls`, `poster` 적용
- [x] 히어로 배경을 gradient 오버레이와 이미지로 겹쳐서 구성
- [x] 명소 카드에 `:hover` 시 이미지가 `scale`로 확대되는 효과 적용

#### 기본 테스트

- [x] 내비게이션의 “먹거리”를 클릭하면 해당 섹션으로 부드럽게 스크롤되는지 확인
- [x] 아래로 스크롤해도 상단 내비게이션이 계속 고정되는지 확인

#### 검증 테스트

- [x] 명소 카드에 마우스를 올리면 이미지가 살짝 확대되는지 확인
- [x] 푸터의 “↑ 맨 위로”를 클릭하면 페이지 최상단으로 이동하는지 확인

#### 결과 확인 테스트

- [x] 이미지 로딩을 차단하거나 개발자 도구로 모든 `img`에 의미 있는 `alt`가 있는지 확인
- [x] 영상 섹션의 재생 버튼으로 재생·정지가 동작하는지 확인(인터넷 필요)

#### 자기평가

- 잘된 점:
  1. `header`, `nav`, `main`, `section`, `article`, `aside`, `footer`를 각 영역의 역할에 맞게 사용하여 페이지의 구조와 의미가 드러나도록 작성한 점이 잘되었다고 생각합니다.
  2. 명소 이미지와 설명을 `figure`와 `figcaption`으로 묶고 모든 이미지에 의미 있는 `alt`를 작성했으며, `video`에 `controls`, `poster`, `source`를 적용하여 미디어 요소를 목적에 맞게 활용한 점이 잘되었다고 생각합니다.
  3. 고정 내비게이션의 링크를 각 섹션의 `id`와 연결하고 부드러운 스크롤과 맨 위로 이동 기능을 구현하여 긴 페이지에서도 원하는 내용을 편리하게 찾을 수 있도록 한 점이 잘되었다고 생각합니다.
  4. 데스크톱, 태블릿, 모바일 화면에 맞게 배치를 조정하고 큰 글씨 모드와 내비게이션의 접근 가능한 이름을 제공하여 다양한 사용 환경을 고려한 점이 잘되었다고 생각합니다.
- 어려웠던 점과 해결 과정:
  1. 고정 내비게이션을 적용한 뒤 화면 크기가 줄어들면 메뉴가 줄바꿈되고 히어로 영역과 겹치는 문제가 있었습니다. 데스크톱, 태블릿, 모바일의 화면 너비에 맞게 내비게이션과 콘텐츠 배치를 조정하면서도 글씨가 지나치게 작아지지 않도록 구성하는 부분이 어려웠습니다.
  2. 정해진 디자인이 없어 페이지의 전체적인 색상과 배치 방향을 정하는 부분이 어려웠습니다. Google Stitch로 참고할 디자인 시안을 만든 뒤 과제 요구사항에 맞게 필요한 부분을 적용하여 디자인 방향을 정할 수 있었습니다.
- 개선할 점:
  1. 홍보 영상에 자막이나 영상 내용을 설명하는 대체 자료가 없어 소리를 듣기 어려운 사용자는 내용을 이해하기 어렵습니다. 자막 파일과 영상 설명을 제공하여 접근성을 보완하고 싶습니다.
  2. 반복되는 명소 카드를 JavaScript로 생성하는 방법도 고려했지만, 아직 컴포넌트 개념을 배우지 않았고 이번 실습은 시맨틱 태그의 의미와 활용을 이해하는 것이 목적이므로 HTML에 직접 작성하였습니다. 추후 컴포넌트 개념을 학습한 뒤 반복되는 영역을 분리하여 재사용과 유지보수가 쉽도록 개선하고 싶습니다.

  1. 데스크톱·태블릿·모바일 화면에 대응하도록 명소 갤러리와 내비게이션에 반응형 디자인을 적용
  2. 추가 관광 정보를 확인할 수 있도록 서울 공식 관광 안내 사이트인 Visit Seoul 외부 링크를 제공
  3. 다양한 연령대의 사용자가 여행 정보를 편하게 읽을 수 있도록 큰 글씨 모드를 제공

- [x] 상단바에서 로고는 왼쪽, 메뉴·버튼은 오른쪽으로 정렬되는지 확인
- [x] OS의 ‘동작 줄이기’를 켜고 새로고침하면 애니메이션이 비활성화되는지 확인

#### 자기평가

- 잘된 점:
  1. CSS Grid와 Flexbox를 목적에 맞게 사용하고, 화면 너비에 따라 카드가 3열, 2열, 1열로 변경되도록 구현하여 반응형 레이아웃을 잘 반영한 점이 잘되었다고 생각합니다.
  2. CSS 변수로 색상을 관리하여 변수값만 교체해 전체 다크 테마가 적용되도록 하고, hover와 등장 애니메이션 및 모션 최소화 설정까지 과제 요구사항에 맞게 구현한 점이 잘되었다고 생각합니다.
  3. 새로고침하면 초기화되던 다크모드 상태를 `localStorage`에 저장하여 사용자가 페이지에 다시 접속해도 선택한 테마가 유지되도록 개선한 점이 잘되었다고 생각합니다.
  4. `loading`, `lang`, `aria-pressed` 등의 속성을 학습하고 적용했으며, 다크모드 상태에 따라 `aria-pressed` 값이 JavaScript로 동적으로 변경되도록 구현하여 웹 성능과 접근성을 함께 고려한 점이 잘되었다고 생각합니다.
- 어려웠던 점과 해결 과정:
  1. OS의 ‘동작 줄이기’ 기능을 처음 접하여 이를 웹 페이지에 어떻게 반영해야 하는지 이해하는 부분이 어려웠습니다. 움직임에 민감한 사용자는 애니메이션으로 인해 어지럼증이나 멀미를 느낄 수 있다는 것을 알게 되었으며, `prefers-reduced-motion`을 사용하여 해당 설정이 활성화된 경우 애니메이션과 전환 효과가 실행되지 않도록 해결하였습니다. 이 과정을 통해 운영체제의 접근성 설정을 웹 페이지에서도 고려해야 한다는 점을 배웠습니다.
- 개선할 점:
  1. 상품 카드가 HTML에 반복되어 있어 상품 정보가 변경되거나 카드가 추가될 때 각각 수정해야 합니다. 이번 실습에서는 다루지 않은 방법이라 적용하지 않았지만, 추후 상품 데이터를 분리하고 JavaScript 함수로 카드를 생성하여 유지보수하기 쉽도록 개선하고 싶습니다.
  2. 기능 구현에 집중하여 카드의 그림자, 글꼴, 여백 등 세부적인 디자인이 아직 단순합니다. 실제 서비스에서 사용할 수 있도록 요소별 디자인과 화면의 완성도를 보완하고 싶습니다.
  3. 현재는 모든 상품 카드가 한 페이지에 표시되므로 상품이 많아지면 페이지가 길어질 수 있습니다. 한 번에 표시할 상품 수를 정하고 페이지를 이동할 수 있는 페이지네이션 기능 또는 무한 스크롤 기능을 추가하고 싶습니다.
  4. 현재는 모든 종류의 상품이 함께 표시되어 원하는 상품을 구분해서 보기 어렵습니다. 상품에 카테고리를 지정하고 필터 기능을 추가하여 사용자가 원하는 종류의 상품만 확인할 수 있도록 개선하고 싶습니다.
- 실습 이후 추가한 과제/기능:
  1. `localStorage`를 활용하여 새로고침 후에도 사용자가 선택한 다크모드 상태를 유지
  2. 초기 이미지 로딩 부담을 줄이기 위해 상품 이미지에 `loading="lazy"`를 적용
  3. 스크린 리더에서도 다크모드 선택 상태를 확인할 수 있도록 버튼에 `aria-pressed`를 적용
  4. 검색엔진과 스크린 리더가 문서의 언어를 확인할 수 있도록 `lang="ko"`를 적용
